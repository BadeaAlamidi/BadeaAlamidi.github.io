Marching Cubes:

As a programmer, I was responsible for establishing a 3D space, reading in threshold values, and using those values as I iterate over the 3D space, voxel by voxel, to assign a deterministic simple shape to the current voxel. Click here if you're interested in learning more (skip to 2nd page)
The voxels, when viewed all together on a 3D - renderer, will represent the mesh that is described in the input file.
This algorithm is used in the medical field to draw three-dimensional surfaces of the patient's internal injury

The big moment in this video is @00:20, when the marchingcubes program is run. It outputs a polyset.rd file for the renderer to display. The input file in this demo, mc1.dat, 3 space-seperated integer values that tell the program the size of the spread 3D array, followed by the values of the 3D array in their hexadeciaml format.

Fractal Terrains

The following is a C++ implementation of Richard Voss' algorithm for creating fractal terrains
The program accepts a seed for randomization and calculates fractal dimensions to generate z values that represent realistic looking terrains when used for a 2D grid.

Using the same aproach for calculating per-vertex normals from marchingcubes, I tried to calculate normal vectors to make the terrain look smoother... I wonder if my eyes are fooling me into thinking that these normals are okay.

Search Feature:

For this project, I've been primarily focusing on implementing a tag-search feature that compliments the schema of the website's database. In my attempt to work with the ORM (sequelize), I also had to put a lot of work into making the feature intuitive for the average user

A challenge arises when filtering books using a toxi solution for the schema (a many to many relationship between books and tags created by a junction table): What if the authors are put not as tags, but are stored in a seperate authors table that has the same kind of relationship between tags and books (many authors also belong to many books).
There is a lot to talk about when it comes to this project. The biggest topics that come to my mind are containerization, normalization, CRUD operations, "REST" api, and that's not to include the process and the practices of front-end design.

Entity Framework REST Demonstration

The following a very simple project that aims to explore the Entity Framework ORM and how it generates the schema I define in the data context file. Unlike Sequelize, Entity Framework was very straight forward and much more user-friendly.

To fully explore EF, this project establishes a few endpoints that test the relationships between the different tables in the database.
Interfacing with the database using EF is explicit in terms of C#, making it easy for the programmer to manipulate and query the database.
My favorite thing about Entity Framework is how good it is at creating the database schema. The picture shows a foreign key relationship in this table's design, and all we needed was to create a field in our model class that has the type of another model. Entity Framework also creates junction tables for many-to-many relationships!
